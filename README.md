# DB Papers

**数据库系统论文中文全文翻译集**

这里持续收录数据库系统领域具有代表性的论文，并为符合当前翻译策略的论文提供中文全文翻译，面向关注数据库内核、查询优化、执行引擎、存储系统和分布式数据库的读者。

译文尽量保留原论文的章节结构、公式、图表、代码和参考文献，既适合连续阅读，也便于与原文对照。

[浏览全部论文](CATALOG.md) · [查询处理](papers/query-processing/) · [系统架构](papers/system-architecture/) · [存储](papers/storage/) · [分布式数据库](papers/distributed-databases/)

## 这里有什么

仓库内容主要覆盖：

- **查询处理**：查询优化、执行模型、查询编译、向量化执行与并行执行
- **系统架构**：云数据仓库、实时分析系统、Serverless 与大型数据平台
- **存储系统**：行列存布局、缓存友好设计、SSD、分布式文件系统与存储引擎
- **分布式数据库**：分布式 SQL、事务、复制、容错与地理分布
- **事务与并发控制**：隔离级别、锁、恢复、内存事务与确定性事务调度
- **AI 与数据系统**：学习型索引、基数估计与优化器，语言模型数据交互和 AI 分析系统
- **向量检索与湖仓**：向量索引、向量数据库、对象存储表格式和湖仓执行引擎
- **数据集成与流处理**：数据导入、格式解析、流批一体与实时计算
- **可靠性与测试**：优化器测试、测试数据生成与基准分析

完整收录范围、论文主题和译文状态请查看 [论文目录](CATALOG.md)。

## 如何阅读

每篇论文位于独立目录中，可能包含：

```text
translation.md   中文全文译文
source.pdf       论文原文
assets/          译文引用的图表和插图
paper.yaml       标题、作者、年份、来源、主题和阅读状态
```

从 [论文目录](CATALOG.md) 点击标题即可进入对应目录。如有译文，可优先阅读 `translation.md`；核对术语、公式、实验数据或引用时，请同时参考 `source.pdf`。

目录中的阅读状态表示：

- `translated`：已提供经过完整性与准确性检查的中文全文译文，可直接点击论文标题阅读
- `draft`：译文尚未完成验收
- `source_only`：仅保存原文，暂未提供译文
- `skipped`：保留论文记录和原文，但当前不提供译文
- `unavailable`：当前没有可读的原文或译文

## 评分说明

目录中的评分衡量论文是否值得优先阅读，不评价译文质量。总分为 1–5 分，以 0.5 为档位，由影响广度、技术价值、实践扩散、持久性和阅读收益五项加权得出；只有原文和外部影响证据充分的论文才会评分，未评分不代表论文价值较低。

## 译文说明

译文以完整、准确和可连续阅读为目标，不以摘要代替正文。重要专业术语首次出现时通常保留英文，公式、图表及其编号尽量与原文保持一致。

翻译难免存在疏漏。用于学术引用、实现验证或精确数据核对时，请以论文原文为准。论文版权及授权以原文和出版平台的标注为准。

如果发现错译、漏译或排版问题，欢迎通过 GitHub Issue 反馈。
